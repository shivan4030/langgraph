## 2026-05-12 - Prevent Credential Leakage in Cross-Origin Redirects
**Vulnerability:** Open Redirect Credential Leakage: Custom reconnect/redirect logic for HTTP and SSE streaming passed the original request headers (including `x-api-key`) to the new location without verifying if the new location was on the same origin as the original request.
**Learning:** `httpx` inherently strips authentication headers for cross-origin redirects, but the custom logic in `libs/sdk-py/langgraph_sdk/_async/http.py` and `libs/sdk-py/langgraph_sdk/_sync/http.py` (which handled redirects manually, especially for SSE fallback paths via `request_reconnect` and inside the `stream` while-loop) neglected this standard security measure.
**Prevention:** Always use URL parsing (like `urllib.parse`) to compare the scheme and network location (`netloc`) between the original request URL and the redirect location. Strip sensitive headers if `is_cross_origin()` is true before executing the sub-request.
